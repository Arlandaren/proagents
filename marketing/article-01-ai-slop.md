## Всё началось с кнопки

Вы открываете Cursor, просите его написать форму регистрации. Модель пишет — и выдаёт:

```jsx
<button className="bg-blue-500 text-white px-4 py-2 rounded">
  Submit
</button>
```

Знакомо? Именно это имеют в виду под термином **AI slop** — технически работающий, но абсолютно безликий, шаблонный код, который выглядит так, будто его списали с первого туториала по Tailwind.

Это не баг модели. Это баг контекста.

---

## Почему ИИ пишет «слоп»

LLM не знает о вашем проекте ничего, пока вы не скажете. Без правил она оптимизирует под **вероятность** — то есть генерирует самое «среднее» из обучающей выборки.

Среднее по GitHub — это именно те серые кнопки. Среднее по StackOverflow — это `console.log` вместо нормального логирования. Среднее по туториалам — это хардкод `SECRET_KEY = "my_secret"`.

**Модели не плохие. Им просто не дали правильный системный промпт.**

---

## Что такое системный промпт и почему он решает всё

Системный промпт — это инструкция, которая выполняется до вашего первого сообщения. Именно там живут:

- стандарты стиля кода
- правила безопасности
- ограничения по архитектуре
- требования к тестированию

Большинство разработчиков либо не используют их совсем, либо пишут что-то вроде `"You are a helpful senior developer"` — что абсолютно бесполезно.

## Встречайте proagents

[**proagents**](https://github.com/Arlandaren/proagents) — открытый репозиторий с **770+ профессиональных системных промптов и воркфлоу**, организованных по функциональным доменам.

Вот что меняется после установки одного правила из библиотеки:

| Аспект | Стандартный ИИ | proagents |
|:---|:---|:---|
| **UI/UX** | Серые Tailwind-кнопки, стандартные шрифты | HSL-палитры, glassmorphism, микроанимации |

| **Производительность** | Лишние ре-рендеры, без оптимизаций | Точечные DOM-операции, LCP-оптимизация |

## Конкретный пример: до и после

**Запрос**: «Напиши компонент карточки продукта на React»

**Без proagents** — классический слоп:
```jsx
function ProductCard({ product }) {
  return (
    <div className="border p-4 rounded">
      <h2>{product.name}</h2>
      <p>{product.price}</p>
      <button className="bg-blue-500 text-white p-2">Buy</button>
    </div>
  );
}
```

**С правилом `ui-taste` из proagents** — модель сама прописывает CSS-переменные, добавляет `aria-label`, оборачивает в `motion.div` с spring-анимацией, валидирует props через TypeScript интерфейс, и использует семантический `<article>` вместо `<div>`.

Разница — один `--cursor` флаг в CLI.

## Как это работает за 2 минуты

```bash
# Ищем нужный домен
python proagents.py search ui

# Устанавливаем в Cursor
python proagents.py install ui-taste --cursor
```

Всё. Файл `.cursorrules` обновится, и следующий запрос к Cursor уже пройдёт через строгий фильтр качества.

Для Claude Code:
```bash
python proagents.py install production-deployment --stdout >> CLAUDE.md
```

## Что внутри библиотеки

Более 770 промптов разбиты на четыре блока:

- **rules/** — глобальные токены: цветовая теория, гигиена кода, git-соглашения
- **personas/** — специализированные роли: SRE, Solidity-разработчик, UX-исследователь, юрист
- **workflows/** — пошаговые воркфлоу: TDD-циклы, WCAG-аудиты, OWASP threat modeling
- **templates/** — готовые шаблоны: ADR, спецификации, структуры проектов
